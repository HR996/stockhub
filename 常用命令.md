# 常用命令

本文档整理本项目 PostgreSQL 数据库的连接、查看、迁移、备份和排错命令。

## 数据库连接信息

本项目通过 `docker-compose.yml` 启动 PostgreSQL 15。

| 项目 | 值 |
| --- | --- |
| 容器名 | `istock-postgres` |
| 数据库类型 | `PostgreSQL 15` |
| 宿主机地址 | `localhost:5432` |
| Docker 网络内地址 | `postgres:5432` |
| 用户名 | `istock` |
| 密码 | `istock` |
| 数据库名 | `istock` |
| 后端容器内 `DATABASE_URL` | `postgresql+psycopg://istock:istock@postgres:5432/istock` |
| 宿主机开发 `DATABASE_URL` | `postgresql+psycopg://istock:istock@localhost:5432/istock` |

## 启动和停止数据库

```bash
# 只启动 PostgreSQL
docker compose up -d postgres

# 启动完整项目：postgres + backend + frontend
docker compose up -d

# 查看服务状态
docker compose ps

# 查看数据库日志
docker compose logs -f postgres

# 停止服务，但保留数据库数据卷
docker compose down

# 停止服务并删除数据库数据卷，谨慎使用
docker compose down -v
```

## 安装 psql 客户端

如果使用 `docker compose exec postgres psql ...` 进入数据库，不需要在宿主机额外安装 `psql`，因为 PostgreSQL 容器里已经自带。

如果想在宿主机直接执行：

```bash
psql "postgresql://istock:istock@localhost:5432/istock"
```

则需要安装 PostgreSQL 客户端。

### Ubuntu / Debian

```bash
sudo apt update
sudo apt install -y postgresql-client

# 验证安装
psql --version
```

### TencentOS / RHEL / CentOS

本机是 TencentOS Server 3.2，属于 RHEL/CentOS 8 系。项目数据库是 PostgreSQL 15，建议安装同主版本的 `psql` 客户端。

```bash
# 查看可用 PostgreSQL 模块
dnf module list postgresql

# 切换到 PostgreSQL 15 客户端
sudo dnf module reset -y postgresql
sudo dnf module enable -y postgresql:15
sudo dnf install -y postgresql

# 验证安装
psql --version
```

如果已经装过旧版客户端，可以执行：

```bash
sudo dnf module reset -y postgresql
sudo dnf module enable -y postgresql:15
sudo dnf distro-sync -y postgresql
psql --version
```

### macOS

使用 Homebrew：

```bash
brew install libpq
brew link --force libpq

# 验证安装
psql --version
```

也可以安装完整 PostgreSQL：

```bash
brew install postgresql@15
brew link --force postgresql@15
psql --version
```

### Windows

推荐安装 PostgreSQL 官方安装包，安装时勾选 Command Line Tools：

```text
https://www.postgresql.org/download/windows/
```

安装后如果命令行找不到 `psql`，把 PostgreSQL 的 `bin` 目录加入 `PATH`。常见路径类似：

```text
C:\Program Files\PostgreSQL\15\bin
```

重新打开 PowerShell 或 CMD 后验证：

```powershell
psql --version
```

### Docker 临时使用 psql

如果不想安装客户端，也可以临时启动一个 PostgreSQL 客户端容器连接宿主机端口。

Linux 常用：

```bash
docker run --rm -it postgres:15 psql "postgresql://istock:istock@host.docker.internal:5432/istock"
```

如果 Linux 上 `host.docker.internal` 不可用，可以加上：

```bash
docker run --rm -it --add-host=host.docker.internal:host-gateway postgres:15 \
  psql "postgresql://istock:istock@host.docker.internal:5432/istock"
```

## 连接数据库

推荐方式：直接进入 PostgreSQL 容器内使用 `psql`。

```bash
docker compose exec postgres psql -U istock -d istock
```

如果宿主机已经安装 `psql`，也可以直接连接宿主机端口。

```bash
psql "postgresql://istock:istock@localhost:5432/istock"
```

也可以用容器运行一次性 `psql` 命令，不进入交互模式。

```bash
docker compose exec postgres psql -U istock -d istock -c "SELECT current_database(), current_user;"
```

## psql 内常用元命令

以下命令在进入 `psql` 后执行。

```sql
-- 查看所有数据库
\l

-- 连接到 istock 数据库
\c istock

-- 查看当前连接信息
\conninfo

-- 查看所有 schema
\dn

-- 查看 public schema 下的表
\dt public.*

-- 查看表、视图、序列等对象
\d

-- 查看某张表结构
\d stock_basic
\d k_line_daily
\d trade_calendar
\d data_update_task
\d latest_market_cap

-- 查看某张表的详细结构，包括索引、约束等
\d+ stock_basic
\d+ k_line_daily

-- 查看所有索引
\di

-- 查看所有序列
\ds

-- 打开更易读的竖排显示
\x

-- 显示执行耗时
\timing

-- 退出 psql
\q
```

## 查看库、表、字段和数据量

```sql
-- 查看当前数据库
SELECT current_database();

-- 查看当前用户
SELECT current_user;

-- 查看 public schema 下所有表
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_type = 'BASE TABLE'
ORDER BY table_name;

-- 查看每张表的估算行数
SELECT
  schemaname,
  relname AS table_name,
  n_live_tup AS estimated_rows
FROM pg_stat_user_tables
ORDER BY relname;

-- 查看指定表字段
SELECT
  column_name,
  data_type,
  is_nullable,
  column_default
FROM information_schema.columns
WHERE table_schema = 'public'
  AND table_name = 'stock_basic'
ORDER BY ordinal_position;

-- 查看索引
SELECT
  tablename,
  indexname,
  indexdef
FROM pg_indexes
WHERE schemaname = 'public'
ORDER BY tablename, indexname;

-- 查看外键、唯一约束、主键等约束
SELECT
  conrelid::regclass AS table_name,
  conname AS constraint_name,
  contype AS constraint_type,
  pg_get_constraintdef(oid) AS definition
FROM pg_constraint
WHERE connamespace = 'public'::regnamespace
ORDER BY table_name::text, constraint_name;
```

## 查看业务数据示例

```sql
-- 股票基础信息
SELECT *
FROM stock_basic
ORDER BY ts_code
LIMIT 20;

-- 按市场统计股票数量
SELECT market, COUNT(*) AS count
FROM stock_basic
GROUP BY market
ORDER BY count DESC;

-- 交易日历
SELECT *
FROM trade_calendar
ORDER BY cal_date DESC
LIMIT 20;

-- 最近 K 线数据
SELECT ts_code, trade_date, close_raw, close_qfq, volume, amount
FROM k_line_daily
ORDER BY trade_date DESC, ts_code
LIMIT 50;

-- 查看某只股票 K 线
SELECT *
FROM k_line_daily
WHERE ts_code = '000001.SZ'
ORDER BY trade_date DESC
LIMIT 30;

-- 最新市值快照
SELECT *
FROM latest_market_cap
ORDER BY snapshot_at DESC
LIMIT 20;

-- 数据更新任务日志
SELECT id, task_type, task_key, status, started_at, finished_at, success_count, error_count, created_by
FROM data_update_task
ORDER BY started_at DESC
LIMIT 20;
```

## Alembic 迁移命令

项目使用 Alembic 管理数据库 schema，迁移文件在 `backend/alembic/versions/`。

宿主机执行迁移时，需要让 `DATABASE_URL` 指向宿主机端口：

```bash
cd backend
export DATABASE_URL='postgresql+psycopg://istock:istock@localhost:5432/istock'

# 查看当前数据库迁移版本
uv run alembic current

# 升级到最新迁移
uv run alembic upgrade head

# 查看迁移历史
uv run alembic history

# 查看当前 head
uv run alembic heads

# 自动生成迁移，生成后要人工检查
uv run alembic revision --autogenerate -m "describe change"

# 回退一个版本，谨慎使用
uv run alembic downgrade -1
```

如果在后端容器里执行迁移，可以使用 Docker 网络内地址：

```bash
docker compose exec backend sh
export DATABASE_URL='postgresql+psycopg://istock:istock@postgres:5432/istock'
uv run alembic upgrade head
```

## 备份和恢复

备份当前数据库到本地文件：

```bash
docker compose exec -T postgres pg_dump -U istock -d istock > istock_backup.sql
```

从备份恢复：

```bash
docker compose exec -T postgres psql -U istock -d istock < istock_backup.sql
```

备份为自定义格式，适合 `pg_restore`：

```bash
docker compose exec -T postgres pg_dump -U istock -d istock -Fc > istock_backup.dump
```

恢复自定义格式备份：

```bash
docker compose exec -T postgres pg_restore -U istock -d istock --clean --if-exists < istock_backup.dump
```

## 清空或重建本地数据库

只清空业务表数据，保留表结构：

```sql
TRUNCATE TABLE
  latest_market_cap,
  k_line_daily,
  stock_basic,
  trade_calendar,
  data_update_task
RESTART IDENTITY CASCADE;
```

完全删除 Docker 数据卷并重建数据库，谨慎使用：

```bash
docker compose down -v
docker compose up -d postgres
cd backend
export DATABASE_URL='postgresql+psycopg://istock:istock@localhost:5432/istock'
uv run alembic upgrade head
```

## Python 快速连库检查

在 `backend` 目录下执行：

```bash
cd backend
export DATABASE_URL='postgresql+psycopg://istock:istock@localhost:5432/istock'
uv run python - <<'PY'
import psycopg

with psycopg.connect("postgresql://istock:istock@localhost:5432/istock") as conn:
    with conn.cursor() as cur:
        cur.execute("""
            SELECT table_name
            FROM information_schema.tables
            WHERE table_schema = 'public'
            ORDER BY table_name
        """)
        for row in cur.fetchall():
            print(row[0])
PY
```

## 常见排错

### 连接被拒绝

先确认数据库容器是否启动并健康：

```bash
docker compose ps
docker compose logs postgres
```

如果没有启动：

```bash
docker compose up -d postgres
```

### 宿主机连接失败，但容器内可以连接

确认端口映射是否存在：

```bash
docker compose port postgres 5432
```

期望能看到类似 `0.0.0.0:5432` 或 `:::5432`。

### 后端容器能连，宿主机不能连

后端容器内使用的是：

```text
postgresql+psycopg://istock:istock@postgres:5432/istock
```

宿主机应使用：

```text
postgresql+psycopg://istock:istock@localhost:5432/istock
```

`postgres` 是 Docker Compose 网络内的服务名，宿主机通常不能直接解析这个名字。

### 查看当前数据库实际迁移版本

```sql
SELECT * FROM alembic_version;
```

如果表不存在，通常说明还没有执行 Alembic 迁移。

### 执行 `\d+ 表名` 报 `column c.relhasoids does not exist`

这是 `psql` 客户端版本太旧导致的兼容性问题，不是业务表损坏。

典型场景：

```text
psql 客户端：10.x
PostgreSQL 服务端：15.x
```

PostgreSQL 12 以后移除了系统列 `relhasoids`，旧版 `psql` 的 `\d+` 仍会查询它，所以会报错。

解决方式：升级本机 `psql` 到与服务端一致的主版本，例如 PostgreSQL 15。

```bash
psql --version
PGPASSWORD=istock psql "postgresql://istock@localhost:5432/istock" -c "SHOW server_version;"

sudo dnf module reset -y postgresql
sudo dnf module enable -y postgresql:15
sudo dnf distro-sync -y postgresql

psql --version
```

升级后重新进入数据库：

```bash
PGPASSWORD=istock psql "postgresql://istock@localhost:5432/istock"
```

再执行：

```sql
\d+ k_line_daily
\d+ stock_basic
\d+ data_update_task
```
